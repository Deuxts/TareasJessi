¿Cuáles son los componentes de las reglas CSS?

Reglas

Se le llama regla a los bloques de códigos que escribimos y dentro de los mismos hay ciertas reglas que debemos respetar

Reglas

  • Selector:
      Es quien comienza la regla y define a quien se aplicara, si a una etiqueca como los es < p> o si sera aplicado a una "clase"

  • Declaración:
      Especifica a cuál de las propiedades del elemento quieres dar estilo.

  • Propiedades:
      En CSS, seleccionas que propiedad quieres afectar en tu regla.

¿Por qué se recomienda el uso de CSS?

  1. Separación del contenido y presentación.
  2. Flexibilidad.
  3. Unificación del diseño de las páginas del sitio.
  4. Optimización de los tiempos de carga y de tráfico en el servidor.
  5. Precisión.
  6. Accesibilidad y estructuración.
  7. Limpieza del código fuente.
  8. Compatibilidad y continuidad.
  9. Estandarización frente a especificaciones propietarias.

¿Qué es cascada CSS?

Significa que el orden de las reglas importa en CSS: cuando dos reglas tienen la misma especificidad, se aplica la que aparece en último lugar en el CSS.


					h1 { 
						color: red; 
					}
					h1 { 
						color: blue; 
					}

En esta situación el h1 aparecerá azul pues fue la ultima orden que se recibió

¿Qué es especificidad CSS?

Es la manera mediante la cual los navegadores deciden qué valores de una propiedad CSS son más relevantes para un elemento y, por lo tanto, serán aplicados. En pocas palabras entre más específicos seamos con nuestros selectores, mayor prioridad tendrán a pesar de que tengamos selectores parecidos. Incluso podemos usar !important para darle mayor prioridad 


						table td    {height: 50px !important;}
						.myTable td {height: 50px !important;}
						#myTable td {height: 50px !important;}
¿Qué es herencia CSS?

La herencia también debe entenderse en este contexto: algunos valores de las propiedades CSS que se han establecido para los elementos padre los heredan los elementos hijo, pero otros no.

¿Qué es padding?

Establece el espacio de relleno requerido por todos los lados de un elemento. El área de padding es el espacio entre el contenido del elemento y su borde (border). No se permiten valores negativos.

ejemplo


						.texto{
							padding: 1em; /*afecta a todos los lados por igual*/
							padding: 10% 0; /* el 10% afecta a los lados de arriba y abajo
							                   y el 0 afecta los lados */
							padding: 10px 50px 30px 0;/* cada lado tiene su propio padding*/
						}
¿Qué es la propiedad margin?

La propiedad margin establece los márgenes de un elemento especificando entre uno y cuatro valores, donde cada valor es una longitud, un porcentaje o auto. Los valores en porcentaje se refieren al ancho del elemento padre. Se permiten márgenes negativos.

Sintaxis 

						/* Aplica a todos los cuatro lados */
						margin: 1em;

						/* Vertical | Horizontal */
						margin: 5% auto;

						/* Arriba | Horizontal | Abajo */
						margin: 1em auto 2em;

						/* Arriba | Derecha | Abajo | Izquierda */
						margin: 2px 1em 0 auto;

						/* Valores globales */
						margin: inherit;
						margin: initial;
						margin: unset;
Ejemplo 

						.ex1 {
							margin: auto;
							background: gold;
							width: 66%;
						}
						.ex2 {
							margin: 20px 0px 0 -20px;
							background: gold;
							width: 66%;
						}

¿Cuál es la diferencia entre class e id?

La diferencia entre una ID y una clase es que una ID se puede usar para identificar un solo elemento, mientras que una clase se puede usar para agrupar más de uno.
